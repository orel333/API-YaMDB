# API-YaMDB
## Описание
- Данный сервис собирает отзывы (Review) пользователей на произведения (Titles). Произведения делятся на категории: "Книги", "Фильмы", "Музыка". Список категорий (Category) может быть расширен администратором (например, можно добавить категорию "Ювелирка"). 
- В сервисе предусмотрены разные наборы разрешений на действия в зависимости от роли пользоателя: администратор, модератор, просто пользователь. Предусмотрен особый тип \- суперпользователь, у которого есть все разрешения вне зависимости от его "витринной" роли.
- Независимо от способа (API-сервис, командная строка, панель администратора) Регистрация пользователя происходит с выдачей ему JWT-токенов access \- refresh JWT-токена для выдачи нового JWT access-токена на случай завершения действия предыдущего. А также новый refresh и access-JWT токен выдается при изменении роли пользователя через API-сервис либо в панели администратора. Это предусмотрено 

### Технологии
- Python 3.7
- Django 2.2.19
- DRF
- JWT
### Запуск проекта в dev-режиме
- Установите и активируйте виртуальное окружение
- Установите зависимости из файла requirements.txt
```
pip install -r requirements.txt
``` 
- В папке с файлом manage.py выполните команду:
```
python3 manage.py runserver
```
### Распределение задач в команде

- Первый разработчик:
пишет всю часть, касающуюся управления пользователями (Auth и Users): систему регистрации и аутентификации, права доступа, работу с токеном, систему подтверждения через e-mail.
- Второй разработчик:
пишет категории (Categories), жанры (Genres) и произведения (Titles): модели, представления и эндпойнты для них.»
- Третий разработчик:
занимается отзывами (Review) и комментариями (Comments): описывает модели, представления, настраивает эндпойнты, определяет права доступа для запросов. Рейтинги произведений тоже достаются третьему разработчику.
